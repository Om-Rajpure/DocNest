# 📌 Project Overview

DocNest is a full-stack enterprise-inspired web application built to simplify client onboarding and document management workflows.

The platform allows organizations to:

* Manage client records
* Upload and organize documents
* Maintain family relationship structures
* Configure onboarding fields dynamically
* Track document validation workflows
* Visualize analytics through dashboards
* Perform CRUD operations efficiently

Unlike traditional academic CRUD projects, DocNest focuses on real-world workflow architecture, scalability, responsive UI, and enterprise-style experience.

# ⚙️ Installation & Setup

## 1️⃣ Clone Repository

```bash
git clone https://github.com/Om-Rajpure/DocNest.git
cd DocNest
```

---

## 2️⃣ Frontend Setup

```bash
cd docnest-frontend
npm install
npm run dev
```

Frontend runs on:

```text
http://localhost:5173
```

---

## 3️⃣ Backend Setup

```bash
cd docnest-backend
mvn spring-boot:run
```

Backend runs on:

```text
http://localhost:8080
```

---

## 4️⃣ Environment Variables

### Frontend (.env)

```env
VITE_API_URL=http://localhost:8080
```

---

## 5️⃣ Database Configuration

Update:

```text
src/main/resources/application.properties
```

Example:

```properties
spring.datasource.url=YOUR_DATABASE_URL
spring.datasource.username=YOUR_USERNAME
spring.datasource.password=YOUR_PASSWORD
```
